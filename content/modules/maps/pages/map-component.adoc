= Map Component

The `GeoMap` UI component displays a geographical data provided by xref:maps:layers-sources.adoc[Layers].

The map is built by superposing layers. Initially, map does not have layers.

You can define the general map parameters along with the layers in the viewâ€™s XML descriptor.

[[map]]
== Map

To use the component in the view, declare the `maps` namespace in the root element of the view's XML descriptor:

[source,xml,indent=0]
----
xmlns:maps="http://jmix.io/schema/maps/ui"
----

Alternatively, you can add the component using Jmix Studio. In this case, the `maps` namespace will be added to the descriptor automatically.

Click *Add Component* in the actions panel, then find the `GeoMap` item and double-click it.

image::adding-map.png[align="center", width="1136"]

The new `geoMap` element will be added in both the *Jmix UI* hierarchy panel and in the XML. You can configure attributes like xref:flow-ui:vc/common-attributes.adoc#id[id], xref:flow-ui:vc/common-attributes.adoc#height[height], xref:flow-ui:vc/common-attributes.adoc#width[width], etc., in the same way as it is done for other xref:flow-ui:vc/components.adoc[UI components].

[source,xml,indent=0]
----
<maps:geoMap id="map"
             height="100%"
             width="100%"/>
----

[[layer]]
== Layer

The `GeoMap` component can include multiple xref:maps:layers-sources.adoc[layers] that display different types of geographic information, such as raster layers, and vector layers.

Initially, map does not have layers.

For example, let's add a xref:maps:layers-sources.adoc#tile-layer[TileLayer] to the map.

Select the `geoMap` element in the *Jmix UI* hierarchy panel or in the view XML descriptor, then click the *Add* button in the inspector panel. From the drop-down list, choose *Layers -> TileLayer*.

image::add-tile-layer.png[align="center", width="331"]

[[source]]
== Source

A xref:maps:layers-sources.adoc[source] represents the data that is used to render map layers on the map component. Sources provide the spatial information that defines the content and appearance of the map layers.

For example, let's add a xref:maps:layers-sources.adoc#osm-source[OsmSource] for the tile layer.

Select `maps:tile` in the *Jmix UI* hierarchy panel or in the view XML descriptor, and then click the *Add* button in the inspector panel. From the drop-down list, choose *OsmSource*.

[[view]]
== View

https://openlayers.org/en/v8.1.0/apidoc/module-ol_View-View.html[View^] defines how the map is displayed in terms of its center, zoom level, rotation, and projection. It essentially sets the initial state of the map when it is loaded in the user interface.

By default, the `geoMap` component displays a world map with an initial geographical center at `(0,0)`.

Choose `geoMap` within the *Jmix UI* hierarchy panel or in the view XML descriptor, and then click the *Add* button in the inspector panel. From the drop-down list, opt for *MapView*.

image::add-map-view.png[align="center", width="372"]

----

You can specify additional parameters:

* centerX defines latitude of the initial geographical center of the map. It is passed to the `org.locationtech.jts.geom.Coordinate` object.
* centerY defines longitude of the initial geographical center of the map. It is passed to the `org.locationtech.jts.geom.Coordinate` object.
* `maxZoom` - sets the maximum zoom level for the view. The default value is 28. For more details see https://openlayers.org/en/v8.1.0/apidoc/module-ol_View-View.html#setMaxZoom[setMaxZoom^].
* `minZoom` - sets a minimum zoom level for the view. The default value is 0. For more details see https://openlayers.org/en/v8.1.0/apidoc/module-ol_View-View.html#setMinZoom[setMinZoom^].
* `projection` refers to the coordinate reference system in which the map is displayed. Common projections include `EPSG:3857 (Web Mercator)` and `EPSG:4326 (WGS 84)`. The default projection is https://epsg.io/4326[EPSG:4326^]. You can set a custom projection in the special inner `projection` element. For more details see https://openlayers.org/en/v8.1.0/apidoc/module-ol_proj_Projection-Projection.html[Projection].
* `rotation` - sets the rotation for the view in radians (positive rotation clockwise, 0 means North). For more details see https://openlayers.org/en/v8.1.0/apidoc/module-ol_View-View.html#setRotation[setRotation^].
* `zoom` defines the specific zoom level.

[[extent]]
== Extent

The https://openlayers.org/en/v8.1.0/apidoc/module-ol_extent.html#~Extent[extent^] specifies the geographic boundaries of the map <<view,view>> or a specific <<layer,layer>> in terms of coordinates, defining the visible area on the map.

Extent is typically defined by minimum (top-left corner) and maximum (bottom-right corner) coordinate values in the form of [`minX`, `minY`, `maxX`, `maxY`], representing the bounding box of the area.

Choose a layer or the `mapView` element in the *Jmix UI* hierarchy panel or in the view XML descriptor, and then click the *Add* button in the inspector panel. From the drop-down list, select *Extent*, and configure the `minX`, `minY`, `maxX` and `maxY` attributes as indicated below.
