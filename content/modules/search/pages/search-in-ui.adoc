= 使用 UI 中的搜索

[[visual-components]]
== 可视化组件

搜索扩展组件提供两个 UI 组件：`SearchField` 和 `FullTextFilter`。

在为项目添加了搜索扩展组件后，这两个 UI 组件可以通过 Studio xref:studio:view-designer.adoc[视图设计器] 的 *Add Component* 工具箱添加。

=== SearchField

`SearchField` 组件提供一个文本框和一个执行搜索的按钮。

视图中使用 `SearchField` 的示例：

----
include::example$/search-ex1/src/main/resources/com/company/demo/view/search/search-view.xml[tags=xmlns;view;search-field]

`SearchField` 组件的属性：

* `searchStrategy` - 定义使用的搜索策略。如未定义，则使用默认策略。
* `entities` - 设置在指定的实体内进行搜索。多个实体可用逗号分隔：
+
[source,xml,indent=0]
----
include::example$/search-ex1/src/main/resources/com/company/demo/view/search/my-search-view.xml[tags=search-field]
----

默认情况下，`searchField` 打开内置的 `SearchResultsView` 视图。该视图会对搜索引擎进行查询，并根据 `searchField` 配置的 `openMode` 的不同在导航页面或对话框内展示搜索结果。

[[full-text-filter]]
=== FullTextFilter

`FullTextFilter` 组件的功能与 xref:flow-ui:vc/components/propertyFilter.adoc[] 类似。

视图中使用 `fullTextFilter` 的示例：

[source,xml,indent=0]
----
include::example$/search-ex1/src/main/resources/com/company/demo/view/search/filter-view.xml[tags=data;layout;fullTextFilter;dataGrid]
----

`FullTextFilter` 组件的属性：

* `dataLoader` - 设置需要过滤的 `DataLoader`。
* `searchStrategy` - 搜索策略。如未定义，则使用默认策略。
* `autoApply` - 如果设置为 `true`，则在组件值发生变更时，组件自动对 `DataLoader` 使用过滤。

`FullTextFilter` 的工作流程：首先使用全文搜索找到实体的 id，然后在数据加载器中添加带这些 ID 的过滤条件。接下来，关联至该加载器的 `DataGrid` 只展示符合全文搜索条件的记录，如果 `DataGrid` 组件有其他过滤器，也会生效。

[[search-strategies]]
=== 搜索策略

搜索策略定义如何处理搜索词。

`SearchField` 和 `FullTextFilter` 组件支持下列搜索策略：

* `anyTermAnyField` - 文档的任何索引字段匹配至少一个输入词。

* `allTermsAnyField` - 文档的任何索引字段以任何顺序匹配上所有的输入词。这是默认策略。

* `allTermsSingleField` - 文档至少有一个字段包含任意顺序的所有输入词。

* `phrase` - 文档至少有一个字段包含输入顺序的所有输入词。

可以使用 `strategy` 属性设置合适的搜索策略，示例：

[source,xml,indent=0]
----
include::example$/search-ex1/src/main/resources/com/company/demo/view/search/filter-view.xml[tags=strategy]
----

如需覆盖默认策略，在 `application.properties` 文件添加下列应用程序属性：

[source,properties,indent=0]
----
include::example$/search-ex1/src/main/resources/application.properties[tags=default-strategy]
----

另外，也可以创建自己的搜索策略。为此，首先创建一个实现 `io.jmix.search.searching.SearchStrategy` 接口的 bean，实现下列方法：

* `String getName()` - 返回策略的唯一名称。

* `void configureRequest(SearchRequest searchRequest, SearchContext searchContext)` - 根据需要配置传入的 `SearchRequest`。

然后，可以在 `SearchField` 或 `FullTextFilter` 组件使用策略名称设置策略。

[[fts-condition-in-filter-component]]
== 过滤器组件中的全文搜索条件

当项目中添加了搜索扩展组件后，xref:flow-ui:vc/components/genericFilter.adoc[GenericFilter] 组件的 *Add condition（添加条件）* 弹窗会出现一个新的条件：

image:search:add_condition.png[align="center",width="1151"]

在 *Full-text filter condition editor（全文过滤条件编辑）* 对话框中，你可以为全文过滤器设置标题并选择一个搜索策略。如果未设置，则使用默认策略。

image:search:condition_editor.png[align="center",width="641"]

与过滤器关联的列表组件中的记录会基于全文搜索结果过滤。
