= 索引队列

在应用程序中添加了搜索扩展组件后，会开始自动 xref:config-options.adoc#tracking-changes[监控实体的修改]。本章节我们介绍如何设置使用最新的数据自动更新索引。

跟踪修改的机制会在 `SEARCH_INDEXING_QUEUE` 数据库表保存对修改后实体操作的队列。`IndexingQueueManager` bean 包含处理队列的方法，并将修改的数据发送至 ES/OpenSearch。

NOTE: 应设置索引队列的定期处理机制，否则搜索将返回过时的数据。

如需实现定期队列处理，可以按照下面的介绍使用 Quartz 任务调度。

[[default-quartz-configuration]]
== 默认 Quartz 配置

搜索扩展组件提供一个预定义的 Quartz 任务配置，用于定期处理索引队列。按下列步骤启用：

. 按照 Quartz 章节的说明在项目中安装 xref:quartz:index.adoc#installation[安装] 扩展组件。

. 按需修改 CRON 表达式。默认值为 `0/5 * * * * ?`（5 秒一次）。
+
[source,properties,indent=0]
----
include::example$/search-ex1/src/main/resources/application.properties[tags=cron]
----

默认配置创建并设置 id 为 `IndexingQueueProcessing` 的调度计划。
